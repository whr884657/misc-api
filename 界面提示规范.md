# misc-api 界面提示规范

本文档定义系统内**操作说明、字段提示、警告提醒**的统一展示方式。后续新增页面请遵循本规范，保持视觉与语义一致。

---

## 1. 组件名称

| 项目 | 说明 |
|------|------|
| CSS 类名 | `.vs-notice`、`.vs-notice--{type}` |
| PHP 函数 | `vs_render_notice($type, $title, $body, $options)` |
| 样式文件 | `assets/css/common.css`（全站可用，含前台 auth 页） |

---

## 2. 提示类型

| type | 用途 | 典型场景 |
|------|------|----------|
| `info` | 一般说明 | 功能介绍、配置说明 |
| `tip` | 字段辅助 | 输入框下方简短说明 |
| `warning` | 需注意 | 更新前备份、邮箱未配置 |
| `success` | 正向反馈 | 可选，操作成功补充说明 |
| `danger` | 禁止/错误 | 严重风险提示 |

---

## 3. PHP 用法

```php
// 区块级提醒（带标题 + 列表）
vs_render_notice(
    'warning',
    '更新前请注意',
    '<ul><li>请先完整备份数据库与网站文件</li><li>点击「安装更新」后将弹出二次确认</li></ul>',
    array('allow_html' => true)
);

// 字段下方简短说明
vs_render_notice('tip', '', '用于找回密码；QQ 邮箱可自动匹配 QQ 头像', array(
    'field' => true,
    'compact' => true,
));

// 带链接（需 allow_html，链接用 vs_e 转义 href）
$settingsUrl = vs_e($vsBase) . '/admin/settings.php';
vs_render_notice(
    'warning',
    '邮箱发信尚未配置',
    '忘记密码功能将不可用。<a href="' . $settingsUrl . '" class="vs-notice__link">前往系统设置</a>',
    array('allow_html' => true)
);
```

### 参数说明

| 参数 | 说明 |
|------|------|
| `$type` | `info` / `tip` / `warning` / `success` / `danger` |
| `$title` | 可选标题，留空则不显示 |
| `$body` | 正文，默认 HTML 转义 |
| `$options['allow_html']` | 为 `true` 时 `$body` 按 HTML 输出（仅用于受控内容） |
| `$options['compact']` | 紧凑间距，适合表单区块内 |
| `$options['field']` | 字段级小提示样式（更小字号、浅底） |

---

## 4. 何时用 Notice，何时用 Alert

| 组件 | 类名 | 场景 |
|------|------|------|
| **Notice** | `.vs-notice` | 静态说明、配置提示、操作前提醒 |
| **Alert** | `.vs-alert` | 表单提交结果、AJAX 动态反馈、错误/成功消息 |

原则：**说明性、预防性**内容用 Notice；**即时反馈**用 Alert 或 Toast。

---

## 5. 已应用页面（1.2.0）

- 系统升级：更新前备份提醒
- 控制台：邮箱未配置警告
- 系统设置：站点字段说明、注册邮箱白名单说明
- 管理员/用户账号设置：头像、邮箱、密码区块提示
- 用户注册：邮箱未配置警告

---

## 6. 新增页面 checklist

- [ ] 是否属于「说明/提醒」而非「操作结果」？→ 使用 `vs_render_notice`
- [ ] 字段下方单行说明？→ `field` + `compact`
- [ ] 含列表或链接？→ `allow_html` + 结构化 HTML
- [ ] 勿再新增裸 `.vs-form-tip` 段落（旧样式逐步替换）
