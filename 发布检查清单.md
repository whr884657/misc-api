## 步骤 1～2：自测与清理

- [ ] 功能自测通过
- [ ] **临时测试目录已删除**（`test/`、`tmp/`、`debug/` 等）→ **不打包、不提交**
- [ ] 临时脚本已删（`*_test.php`、`verify-zip.php` 等）
- [ ] 无 `config/database.php`、`data/`、`upload/`、`release/*.zip` 待提交
- [ ] 无根目录 stray `.zip`
- [ ] `git status` 干净或仅含应提交文件
- [ ] 改动 PHP 已 `php -l`

## 步骤 3：版本与文档（必查）

- [ ] 已判断：**小版本** `1.0.x+1` 或 **大版本** `1.1.0` / `2.0.0`
- [ ] `core/version.php`
- [ ] `update.json`
- [ ] `update-log.json`（数组头部追加）
- [ ] `发行说明/misc-api1.0.NN.md`
- [ ] **`README.md` 已针对本版本更新**（当前版本号、本版特性/变更、目录或能力说明）
- [ ] **仓库简介已检查**（系统能力变动时同步优化 Gitee 描述）
- [ ] 有库表变更 → `install/migrations/1.0.NN.sql` + `db_changes: true`

## 步骤 4～5：推送 Git

- [ ] `git commit`
- [ ] `git push` → `main`
- [ ] `git tag v1.0.NN`
- [ ] `git push` 标签

## 步骤 6～7：打包与本地验 ZIP

- [ ] 使用文档**第五节固定命令**（robocopy → TEMP → Compress-Archive）
- [ ] 输出：`release/misc-api1.0.NN.zip`
- [ ] ZipArchive `open=true`，numFiles 正常
- [ ] 确认 ZIP 内**无测试目录、无 .git、无 release/*.zip**

## 步骤 8～9：Release 与附件

- [ ] 创建 Release（tag = `v1.0.NN`）
- [ ] **`curl.exe -F "file=@..."` 上传**（禁止 PowerShell 手工 multipart）
- [ ] 返回 JSON 的 `size` = 本地字节数
- [ ] 远程下载 ZIP，大小一致且能 ZipArchive 打开
- [ ] （建议）从上一版站点在线升级测试

---

## 打包决策速查

| 内容 | 进 ZIP？ | 进 Git？ |
|------|----------|----------|
| admin / core / assets / install | 是 | 是 |
| 临时测试目录 | **否**（先删） | **否** |
| data/、upload/、database.php | 否 | 否 |
| release/*.zip | 否 | 否 |
| 参考程序/、参考项目/ | 否 | 否 |

---

## 禁止事项

| 禁止 | 原因 |
|------|------|
| 换一套打包/上传命令 | 易导致 ZIP 损坏 |
| PowerShell ISO-8859-1 上传 ZIP | ZIP 损坏 |
