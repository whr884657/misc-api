# 本地接口 · 调用统计代码怎么用

面向站长 / 接口作者。把下面这段代码贴到**本地接口 PHP 文件最上面**（`<?php` 之后、业务代码之前），即可自动统计调用次数并写入日志。

---

## 一、推荐写法（任意目录深度，PHP 7.4～8.2）

```php
<?php
/* 调用统计：请放在业务代码最前面 */
$__d = __DIR__;
while ($__d !== '' && $__d !== dirname($__d)) {
    if (is_file($__d . DIRECTORY_SEPARATOR . 'core' . DIRECTORY_SEPARATOR . 'bootstrap.php')) {
        require_once $__d . DIRECTORY_SEPARATOR . 'core' . DIRECTORY_SEPARATOR . 'bootstrap.php';
        break;
    }
    $__d = dirname($__d);
}
unset($__d);
if (class_exists('ApiStats', false)) {
    ApiStats::hit();
}
```

说明：

1. **不用数目录层数**：无论文件在 `api/`、`api/foo/` 还是 `api/a/b/c/`，都会自动向上找到站点根目录里的 `core/bootstrap.php`。
2. **兼容 PHP 7.4～8.2**（不依赖仅高版本才有的语法）。
3. 后台「接口列表」里该接口的**调用地址**请写成与脚本一致的路径，例如：`/api/yiyan/v1.php`，这样 `ApiStats::hit()` 才能自动对上号。
4. 也可以写成 `ApiStats::hit(12);`（括号里换成后台显示的接口 ID）。

## 二、粘贴位置示例

```php
<?php
/* ↓↓↓ 统计代码贴在这里 ↓↓↓ */
$__d = __DIR__;
while ($__d !== '' && $__d !== dirname($__d)) {
    if (is_file($__d . DIRECTORY_SEPARATOR . 'core' . DIRECTORY_SEPARATOR . 'bootstrap.php')) {
        require_once $__d . DIRECTORY_SEPARATOR . 'core' . DIRECTORY_SEPARATOR . 'bootstrap.php';
        break;
    }
    $__d = dirname($__d);
}
unset($__d);
if (class_exists('ApiStats', false)) {
    ApiStats::hit();
}
/* ↑↑↑ 统计代码结束 ↑↑↑ */

// 下面才是你的业务逻辑
header('Content-Type: application/json; charset=utf-8');
echo json_encode(array('code' => 1, 'msg' => 'ok'));
```

---

## 三、代理接口要不要贴？

**不要。**  
对外地址是 `/apis/短码` 的「代理外链」接口，由系统网关自动统计，**不要**在上游别人的网站文件里贴这段代码。

---

## 四、常见问题

| 现象 | 处理 |
|------|------|
| 调用了但次数不涨 | 后台结构更新是否完成；调用地址是否与脚本路径一致；接口是否已审核通过且未禁用 |
| 提示接口维护中 / 不可用 | 后台把该接口设为正常，并确认已审核通过 |
| 拷贝后语法报错 | 确认文件以 `<?php` 开头，且本段在业务代码之前 |
